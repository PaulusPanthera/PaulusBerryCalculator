What this project structure is for

A small static app that stays easy to host and easy to understand, while still teaching solid habits: structure, naming, formatting, linting, changelogs, releases, and safe separation of content, styles, scripts, and data.

Core ideas

  • Keep deployment simple and static for as long as possible.
  • Keep logic modular: one small file per concern.
  • Keep Git history readable with clear, focused commits.
  • Keep data and calculations separated from UI markup.

Next steps

Replace the sample dataset with your real berry inputs, define your first real calculation rules, and expand the JavaScript modules only when the feature set genuinely needs it.
